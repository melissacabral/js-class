Event Basics

Events are things that happen at a certain time. JavaScript can react to these events. They're happening all the time, whether we react to them or not.

Examples:

  • A button was clicked
  • A key was pressed
  • An input field changed
  • The HTML page finished loading
  • The window was resized

Check the documentation for many examples of events that JS can respond to

Handling Events

Notice that in all of those examples, there is an object that is connected to the event. This will help us write the JS code in the right order

Using onClick

myButton.onclick = function() {
	console.log('you clicked the button');
};

using addEventListener

//object.addEventListener("event", function);
myButton.addEventListener('click', function() {
	console.log('you clicked the button');
});

The Event Handler Function

Taking care of an event means setting up two mechanisms: the event listener and the event handler.

The Event handler is a function that describes "what to do when the event is heard"

In the following code examples, notice there are many valid ways to setup the same functionality:

Callback Function

myButton.addEventListener('click', doTheThing);
function doTheThing(){
	console.log('you clicked');
}

Anonymous Function

myButton.addEventListener('click', function(){
	console.log('you clicked');
});

Arrow Function

myButton.addEventListener('click', () => {
	console.log('you clicked');
});

The Event Parameter

We can tell a lot about the event using an optional parameter in the event function. 

myButton.addEventListener( "click", function(e) {
	//e stands for event data 
	//e.target is the element that was clicked on 
	console.log(e.target); 
 });
